HTML (от англ. HyperText Markup Language — «язык гипертекстовой разметки») — язык разметки, который используется для создания и структурирования веб‑страниц. Он помогает определять, как содержимое страницы должно отображаться в браузерах.

HTML работает по принципу дерева элементов: каждый тег может содержать вложенные теги, образуя иерархию. Современный HTML5 добавил семантические теги вроде header, nav, article и footer, которые делают код более понятным для поисковых систем и скринридеров.

CSS (Cascading Style Sheets, «каскадные таблицы стилей») — язык стилей, с помощью которого оформляют веб‑страницы. Он определяет, как должны отображаться элементы HTML, включая макет, цвета, шрифты и другие визуальные аспекты.

Стили пишутся в виде правил: селектор (например, p или .class) указывает на элемент, а свойства вроде color: red; или font-size: 16px; задают внешний вид. Каскадность означает, что стили применяются с учетом приоритета и порядка.

CSS подключается тремя способами: внутри HTML-тега style, в отдельном файле (через link) или прямо в атрибуте style. Современные возможности включают Flexbox и Grid для сложных макетов, анимации (@keyframes) и медиа-запросы (@media) для адаптивного дизайна под разные экраны. Это позволяет создавать responsive-сайты, которые идеально выглядят на телефонах, планшетах и ПК без переписывания кода.

JavaScript (JS) — язык программирования, который изначально был создан для добавления интерактивности на веб‑страницы.

Он позволяет реагировать на действия пользователя, такие как клики, ввод текста или прокрутка, динамически изменяя содержимое без перезагрузки страницы. JS выполняется в браузере на стороне клиента и использует синтаксис, похожий на C: переменные (let, const), функции, циклы и условия.

Сегодня JavaScript вышел за рамки браузера благодаря Node.js, который позволяет запускать его на сервере. Фреймворки вроде React, Vue или Angular упрощают создание сложных приложений (SPA). JS взаимодействует с HTML/CSS через DOM (Document Object Model) — моделью страницы, где каждый элемент доступен как объект. Например, document.getElementById('id').style.color = 'blue'; мгновенно меняет цвет элемента.

Поддерживает такие стили:

  1. объектно-ориентированный;
  2. императивный;
  3. функциональный.

JavaScript обычно используется как встраиваемый язык для программного доступа к объектам приложений.

Интеграция HTML, CSS и JavaScript формирует полноценное веб-приложение. HTML задаёт каркас, CSS отвечает за дизайн и отзывчивость, а JS реализует логику и пользовательский опыт. Файлы подключаются просто: link rel stylesheet href style.css в head для CSS и script src script.js в конце body для JS. Инструменты разработчика в браузере F12 помогают отлаживать код в реальном времени, проверяя стили, консоль ошибок и сетевые запросы. Такой подход позволяет создавать быстрые и удобные сайты.

Flexbox и Grid в CSS революционизировали создание layouts. Flexbox работает с одномерными последовательностями элементов, где justify-content управляет горизонтальным выравниванием, а align-items вертикальным. Grid подходит для двумерных сеток, с свойствами grid-template-columns и grid-gap. Эти инструменты заменяют старые float и таблицы, делая код проще и гибче. Для мобильных версий комбинируй их с viewport meta-тегом в HTML и breakpoint в медиа-запросах.

Работа с DOM в JavaScript позволяет изменять страницу на лету. querySelectorAll возвращает коллекцию элементов, getElementById находит по id. Методы вроде innerHTML вставляют новый контент, classList.add или toggle управляют классами для стилей CSS. Локальное хранение localStorage.setItem сохраняет данные в браузере, а события вроде load или resize помогают адаптировать поведение. Это основа для SPA-приложений, где весь контент обновляется динамически без серверных запросов.

Адаптивный дизайн или responsive web design делает сайты удобными на любых устройствах. В HTML используй семантические теги вроде header, nav, main, section и footer для лучшей структуры и SEO. В CSS применяй относительные единицы вроде em, rem или процентов вместо фиксированных пикселей, viewport meta-тег в head обеспечивает правильное масштабирование. Медиа-запросы at media (max-width: 768px) меняют стили для планшетов и телефонов, а mobile-first подход начинает с базовых стилей для маленьких экранов, добавляя сложность для больших. JavaScript может проверять window.innerWidth и подгружать разные ресурсы динамически.

Анимации и переходы улучшают пользовательский опыт без перегрузки страницы. В CSS transition: all 0.3s ease плавно меняет свойства вроде opacity или transform при hover или focus. Keyframes определяют сложные анимации at keyframes fadeIn { from { opacity: 0; } to { opacity: 1; } }, применяемые через animation: fadeIn 1s forwards. JavaScript requestAnimationFrame обеспечивает 60 fps для плавности, а IntersectionObserver отслеживает видимость элементов для ленивой загрузки изображений. Эти техники делают сайты современными, но избегай излишеств, чтобы не замедлять производительность на слабых устройствах.

Формы в веб-разработке собирают данные от пользователей и являются ключевым элементом интерактивных сайтов. В HTML создавай их с тегом form, внутри input разных типов text, email, password, number, textarea для многострочного текста, select для выпадающих списков и button или input type submit для отправки. Атрибуты name и id обязательны для идентификации, required обеспечивает валидацию placeholder подсказывает ввод. CSS стилизует поля с border-radius для скругления, focus псевдокласс подсвечивает активные элементы. JavaScript проверяет данные на клиенте через addEventListener submit, preventDefault останавливает отправку при ошибках, а FormData объект собирает все значения для AJAX-запроса через fetch на сервер без перезагрузки страницы.
